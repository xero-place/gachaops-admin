# UI デザイン解説

## 目的

業務オペレーターが日次で利用する管理画面。視覚的に「目立つ」より、データ密度・視認性・操作効率を最優先します。

## デザインの原則

### 1. ダーク優先

オペレーターは早朝〜深夜まで画面を凝視するため、ダークテーマがデフォルトです。`<html className="dark">` を強制適用しており、ユーザがライトに切り替えるオプションは設定画面で提供しています。

### 2. ドメイン色を分離

shadcn の汎用 `primary/destructive` だけだと **業務上のステータス**を表現しきれません。`globals.css` で次のトークンを追加しました:

```css
--ok        /* オンライン、決済成功、配信完了 */
--warn      /* 保守中、低在庫、一部成功 */
--primary   /* PayPay風 vermillion red、CTA とアクティブ状態 */
--destructive /* オフライン、決済失敗、削除 */
```

ステータスバッジ (`src/components/domain/status-badges.tsx`) はこれらを使い分けてアプリ全体で一貫した色語彙を作っています。

### 3. 数値の桁を揃える

すべてのテーブル・KPI カードで `font-variant-numeric: tabular-nums` を強制。¥1,234 と ¥45,678 が縦に並んだとき、桁が揃うことで読みやすさが大きく変わります。

### 4. 情報密度を上げる

- フォントサイズはタイトルでも `text-base` (16px)、テーブル行は `text-xs` (12px)。
- パディングは `p-3` を基本とし、`p-6` は Card のヘッダーだけに使う。
- スクロールバーは細く (10px)、トラックは透明、サムは `--border` カラー。

### 5. 無理な装飾を避ける

エンタープライズダッシュボードで **アニメーション** や **グラデーション** を多用すると、視認性が落ちて疲労感が増します。次のものに限定:

- `live-dot` パルス (オンライン端末のみ)
- 数値の登場時 fade-in (recharts デフォルト)
- カードの hover 状態 (微かな ring/border)

## レイアウト構造

```
┌─────────────────────────────────────────────┐
│ Sidebar (240px)│ Header (56px)              │
│  - ロゴ        ├─────────────────────────────┤
│  - グループ1   │ Main content                │
│  - グループ2   │  - Stats strip (KPIカード)  │
│  - グループ3   │  - Filter bar (Card)        │
│  - グループ4   │  - Data area (Card + Table) │
│                │                              │
│  接続状態      │                              │
└─────────────────────────────────────────────┘
```

### Sidebar

- 4 グループ × 17 リンク (フラット階層を維持)
- アクティブリンクは `bg-primary/15 text-primary` で控えめにハイライト
- フッターでモック/本番モードを表示

### Header

- 左: 画面タイトル + パンくず (text-[11px])
- 右: 検索ボックス (⌘K ヒント) + ベル + ユーザアバター

### Main

3 段構成:
1. **Stats strip** — クリック可能な KPI カード (絞込トグルを兼ねる)
2. **Filter bar** — 検索 / セレクト / 切替トグル
3. **Data area** — Card 内に Table または グリッド

## コンポーネント方針

shadcn/ui を素として、ドメイン特化のラッパーを `src/components/domain/` に置きます。

- `DeviceStatusBadge` → ステータス→色→ラベルの一括変換
- `PlayModeBadge`, `TaskStatusBadge`, `OrderStatusBadge` も同様

これにより、テーブルの個別セルで`<Badge variant=...>` を書く必要がなく、全画面で一貫した表現になります。

## チャート

`recharts` を使用。色は CSS 変数 (`hsl(var(--primary))`) を参照し、テーマ切替に追従できる設計です。グラデーション fill は売上 (面の量感) を直感的に伝えるために使用。

## アクセシビリティ

- `lang="ja"` を root に指定
- フォーカスリング (`focus-visible:ring-2 ring-ring`) を全ボタンで維持
- アイコンのみのボタンは `<span className="sr-only">` を併設 (Dialog の閉じるボタンなど)
- 色だけに依存せず、ステータスは必ずテキストラベルも併記
